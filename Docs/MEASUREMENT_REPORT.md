# Measurement Report

## 1. Overview

This report documents the methodology used to estimate real-world object dimensions from images using a calibrated computer vision pipeline. The system performs camera undistortion, instance segmentation using Mask R-CNN, and pixel-to-millimetre conversion to estimate object width and height.

The objective is to provide an end-to-end measurement workflow capable of processing a single image and returning:

* Segmentation mask overlay
* Object width (mm)
* Object height (mm)
* Detection confidence score

---

## 2. Measurement Methodology

### 2.1 Camera Calibration

Camera intrinsic calibration was performed using a checkerboard calibration target. The resulting camera matrix and distortion coefficients were used to correct lens distortion before any measurements were computed.

Calibration outputs:

* Camera Intrinsic Matrix (K)
* Distortion Coefficients (D)

All measurement images are first undistorted using OpenCV's calibration pipeline.

---

### 2.2 Object Segmentation

Instance segmentation was performed using a Mask R-CNN model trained on a custom labelled dataset.

Model architecture:

* Backbone: ResNet-50 + FPN
* Framework: Torchvision Mask R-CNN
* Classes:

  * Background
  * Coin Object

The model predicts:

* Bounding boxes
* Segmentation masks
* Confidence scores

---

### 2.3 Pixel-to-Millimetre Conversion

After segmentation, the binary mask of each detected object is extracted.

Object dimensions are computed using:

1. Horizontal pixel extent
2. Vertical pixel extent

A pixel-to-millimetre scale factor derived during calibration is applied:

Measurement (mm) = Measurement (pixels) × Scale Factor

This produces:

* Width (mm)
* Height (mm)

for every detected object.

---

## 3. Calibration Dependency

Accurate metric measurement requires image undistortion before segmentation and measurement.

Raw images contain radial and tangential lens distortion that causes non-uniform scaling across the image plane. Objects located near image boundaries may appear stretched or compressed relative to their true dimensions.

To ensure geometric consistency:

1. Input image is undistorted.
2. Segmentation is performed on the undistorted image.
3. Measurements are computed only from undistorted masks.

This guarantees a consistent pixel-to-world mapping throughout the image.

---

## 4. End-to-End Pipeline

The measurement workflow is:

Raw Image
→ Camera Undistortion
→ Mask R-CNN Segmentation
→ Binary Mask Extraction
→ Pixel-to-MM Conversion
→ Width / Height Estimation
→ Annotated Output

Outputs generated by the system:

* Undistorted image
* Segmentation mask overlay
* Width estimate (mm)
* Height estimate (mm)
* Detection confidence

## 6. Accuracy Validation

Accuracy validation should be performed by comparing system-generated measurements against physical ground-truth measurements obtained using a ruler or digital calliper.

For each test object:

1. Measure true width and height manually.
2. Run the image through the pipeline.
3. Record predicted width and height.
4. Compute absolute and percentage errors.

Recommended evaluation metrics:

### Mean Absolute Error (MAE)

MAE = Mean(|Predicted − Ground Truth|)

### Mean Percentage Error (MPE)

MPE = Mean((|Predicted − Ground Truth| / Ground Truth) × 100)

A minimum of 10 independent objects should be evaluated to establish measurement reliability.

---

## 7. Limitations

The current system has several limitations:

* Measurement accuracy depends on calibration quality.
* Incorrect camera positioning may introduce perspective error.
* Partial occlusions can reduce segmentation quality.
* Extremely small objects may be difficult to measure accurately.
* Pixel-to-mm conversion assumes a fixed imaging setup and working distance.

Future improvements may include:

* Perspective correction using homography estimation.
* Automated scale estimation.
* Multi-view measurement.
* Sub-pixel contour refinement.

---

## 8. Conclusion

A complete computer vision measurement pipeline was developed combining camera calibration, instance segmentation, and metric dimension estimation. The system accepts a single image as input and produces segmentation overlays together with real-world width and height measurements. By enforcing image undistortion prior to measurement, the pipeline maintains geometric consistency and provides a foundation for accurate real-world object measurement.
